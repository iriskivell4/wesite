Boilerplate Every web page includes the following parts: a doctype tag an html element a head element a title element a body element Doctypes All HTML documents require a doctype to tell the browser which version of HTML we are using. It must be the first line in the file. Without it, browsers will still show the document but may have unexpected results. The html Element The html tag is always the first tag in the document. It is the root or parent element. All other elements are stored or “nested” inside. The lang attribute classifies the language that the content will be in. Many lang values are available like fr for French or es for Spanish. The head Element The head element includes information about the page, but does not display content. Things like CSS, JavaScript, and metadata can be included here. The is the only required tag inside of this element. The title Element <!DOCTYPE html> <html lang="en"> <head> <title> The title element lives inside the element. It sets the title of the page which is displayed at the top of the browser tab. Always include this in the . The meta Element The meta tag is used to set “metadata” (data about data) for a web page. There a number of uses for meta tags depending on the website. They live inside the element of a web page. The charset attribute defines the version of Unicode that we wish to use. UTF-8 is a common standard for handing the characters used in the world’s writing systems. Always include this in the to avoid character display issues. The body Element All content to be displayed in the browser goes in between the and tags. It is also known as the “viewport”. This is where we will be spending most of our time.
